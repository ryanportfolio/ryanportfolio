# Plan 0016: viewer redesign in the corewise.academy visual language

**Goal:** owner request: the audit viewer at audit.corewise.academy should
share the corewise.academy aesthetic instead of the generic dark-dashboard
look it launched with.

## Design source

Tokens lifted from the live corewise.academy stylesheets (fetched this
session): paper palette (#f3efe4 / ink #1c1914 / accent #2743d0), night
palette (#121420 / #e8e4d8 / #96a8ff), Alembic Titling display font,
Newsreader body serif, IBM Plex Mono uppercase micro-labels with .14em
tracking, thin hairline rules, sharp corners, film-grain overlay, star
glyph accents.

## Scope (this PR)

- `app/src/site/generate.ts`: replace the CSS block and page chrome.
  - Fonts load cross-origin from corewise.academy (owner's own origin;
    `Access-Control-Allow-Origin: *` verified on /fonts/). Fallback stacks
    (Georgia serif, ui-monospace) keep the site readable if fonts fail.
    This amends the file's old "no external assets" note to "no
    third-party assets"; generated bytes stay deterministic.
  - Light theme default, night via `prefers-color-scheme: dark` (no JS;
    the viewer stays zero-script).
  - Masthead wordmark is the literal domain (no invented product name),
    nav links to corewise.academy and the repo.
  - Grade chips and score bars restyled to the editorial palette; grade
    semantics and all copy unchanged.
- No content, scoring, or data changes. Tests asserting content still pass
  unchanged.

## Verification

- `npm test` + `npm run typecheck` green. Determinism is argued by code
  inspection (no clock, network, randomness, or unordered iteration feeds
  the generated bytes); the double-render equality test is retained but is
  not the evidence. Chrome invariants (font origin, fallback stacks, zero
  scripts) get explicit assertions.
- Local `npm run build:site` + browser screenshot check of index and one
  report page, light and dark.
- Post-merge: audit.corewise.academy serves the new design.
